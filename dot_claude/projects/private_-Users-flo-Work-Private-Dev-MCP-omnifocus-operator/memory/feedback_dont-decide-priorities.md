---
name: Don't decide priorities
description: Never tell the user what's low/high priority — recommend the best approach, but leave scheduling and priority to them
type: feedback
---

Don't assign priority labels to suggestions (e.g., "I'd call that low-priority"). The user decides what's important and when to do it.

**Why:** User is explicitly in charge of priorities and sequencing. Assigning priority is a form of steering that oversteps the role.

**How to apply:** Share opinions on *what* is the better approach and *why*. Never comment on *when* or *whether* it's worth doing — that's the user's decision.
